# PET SUVR Scripts

Scripts for co-registering PET scans to MRI and computing SUVRs (Standardized Uptake Value Ratios) by brain region. These are commands run in the container built from the Dockerfile in this repository.

---

## Scripts

| Script | Description |
|--------|-------------|
| `mri_pet_process.sh` | PET-to-MRI registration and SUVR computation driver |
| `petSUVR.R` | R script for SUVR calculation by brain region |
| `mergeTables.py` | Merges per-subject SUVR CSVs into combined tables |
| `runin-maker.py` | Generates `run.in` file for `mri_pet_process.sh` |
| `dicomconvert.sh` | Converts DICOM PET files to NIfTI using `dcmunpack` |
| `FreesurferLUTR.txt` | FreeSurfer lookup table for R (ROI IDs and names) |
| `compoundLU.csv` | Maps subject IDs to PET tracer compound names |

## Filename Convention

PET NIfTI files should follow the pattern:

```text
# e.g. MTS-<group>-<subjid>-<scan_date>-<process_date>_PET_3mmblur.nii
```

MRI subject folders (FreeSurfer output) follow:

```text
# e.g. MTS-<group>-<subjid>-<scan_date>-<process_date>
```

`runin-maker.py` matches PET to MRI by comparing the 3rd `-`-delimited field of each filename (the numeric subject ID).

## Pipeline Steps

### Step 1: Generate run.in

```bash
python3 runin-maker.py
```

Scans `./MRI/` and `../nii/`, matches PET to MRI by subject ID, and writes `run.in`:

```text
<PET_filename_no_ext>,<MRI_subject_folder>
# e.g.
MTS-<group>-<subjid>-<scan_date>-<process_date>_PET_3mmblur,MTS-<group>-<subjid>-<scan_date>-<process_date>
```

Unmatched scans are flagged in `runin-maker.log`. You can also write `run.in` manually.

### Step 2: Run PET processing

```bash
bash mri_pet_process.sh
```

For each `run.in` entry, this script:

1. Converts MRI `T1.mgz` and `aparc+aseg.mgz` to NIfTI in `./temp/` (skipped if already present)
2. Extracts `aparcstatsVolumLeft/Right.csv` and `asegVolume.csv` per subject into `./temp/`
3. Registers the PET to MRI using FSL FLIRT (12 DOF, corratio cost, trilinear interpolation)
4. Calls `petSUVR.R` to compute per-ROI SUVRs; results written to `./res/`

The registered PET (`.nii`) and registration matrix (`.mat`) are saved in `./temp/`.

### Step 3: Prepare compoundLU.csv (optional)

By default all subjects are assumed to use the `Neuraceq` tracer for Centiloid conversion. To specify different tracers, create a `compoundLU.csv` with two columns (no header): subject ID and compound name.

```csv
<subject_ID>,<compound>
# e.g.
<subjid0>,Neuraceq
<subjid1>,Amyvid
```

Supported compounds: `Amyvid`, `Neuraceq`, `PiB`. Subjects not listed default to `Neuraceq`.

### Step 4: Merge SUVR tables

```bash
python3 mergeTables.py
```

Reads `./res/`, merges per-subject files into combined CSVs in `./stats/`, and appends Centiloid values:

| Tracer    | Centiloid formula              |
|-----------|--------------------------------|
| Amyvid    | `183.07 Г— SUVR в€’ 177.26`       |
| Neuraceq  | `153.4 Г— SUVR в€’ 154.9`         |
| PiB       | `100 Г— (SUVR в€’ 1.009) / 1.067` |

## SUVR Output Files

Per-subject results are written to `./res/`. Output filenames follow the pattern `<PET>_reg_<MRI>_<suffix>`.

| File suffix | Description |
|-------------|-------------|
| `_suvr_cerebellum.csv` | Per-ROI SUVR (whole cerebellum reference) |
| `_suvr_cerebellum_gm.csv` | Per-ROI SUVR (cerebellum GM-only reference) |
| `_suvr_combined_cerebellum.csv` | Regional composite SUVRs (frontal, temporal, parietal, cingulate, global) |
| `_suvr_combined_cerebellum_gm.csv` | Same composites, GM-only reference |
| `_mean_suv.csv` | Mean SUV per ROI |
| `_median_suv.csv` | Median SUV per ROI |
| `_total_suv.csv` | Total (summed) SUV per ROI |
| `_max`, `_min` | Global PET max/min voxel values |
| `_suv_cer`, `_suv_cer_gm` | Reference region mean SUV values |
